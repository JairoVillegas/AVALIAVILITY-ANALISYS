# Proyecto: Rappi Makers - Análisis de Disponibilidad de Tiendas

## 🎯 Objetivo General
Construir una aplicación web completa con dos componentes principales (dashboard visual y chatbot semántico) que permita transformar los datos históricos de disponibilidad de tiendas en algo útil, claro y accionable para los clientes.

## 🚀 Arquitectura y Fases del Proyecto

El desarrollo se divide en tres bloques principales para asegurar un ciclo de vida de datos robusto y un producto final excepcional:

- **[Fase 1: Analítica de Datos (BLOQUE_1)](Fase_1_Analitica.md)**
  Procesamiento inicial utilizando CRISP-DM. De crudo a dataset estructurado.
- **[Fase 2: Dashboard Web (BLOQUE_2)](Fase_2_Dashboard.md)**
  Aplicación UI interactiva para visualizar las métricas resultantes y poder extraer insights de negocio.
- **[Fase 3: Agente Semántico (BLOQUE_3)](Fase_3_Chatbot.md)**
  Bot conversacional que responde dinámicamente mediante procesamiento de lenguaje natural integrándose a la web y a los datos.

## 🛠 Entorno y Datos
- **Entorno Virtual:** Se hace uso del entorno virutal `RAPPI_VENV` para separar bibliotecas y dependencias.
- **Datos Originales:** Histórico de cambios en `DATOS/DATOS_CRUDOS/AVAILABILITY-data.csv`.
