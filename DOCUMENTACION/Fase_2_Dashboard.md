# Fase 2: Aplicación Web y Dashboard Interactivo

**Ubicación de Trabajo:** `BLOQUE_2`

##  Objetivo
Desarrollar una aplicación web para que el cliente final pueda ingerir el output de la Fase 1 permitiéndole obtener, filtrar y visualizar profundamente los insights del comportamiento histórico de disponibilidad.

##  Componentes Principales
- **Visualización de Datos:** Gráficas, líneas de tiempo, mapas (si aplica) interactivos.
- **Métricas Relevantes (KPIs):** Indicadores de performance y fallos a nivel general y por tienda.
- **Filtros Flexibles:** Capacidad para el usuario de segmentar el dataset visual por fecha, zonas o tipos de errores de disponibilidad.

##  Características Técnicas Esperadas
1. Ingesta fluida de los datos provenientes de la FASE 1.
2. Interfaz clara y moderna orientada a la experiencia de usuario (UX/UI).

##  Entregable Esperado
Una **aplicación web operativa y atractiva** donde viven los dashboards con gráficos interactivos y métricas derivadas del análisis de datos de los CSV históricos.
